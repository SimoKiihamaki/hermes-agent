## How It Works

### Execution Flow

```
User Request
    ↓
delegate_task(goal="...", task_type="autodev")
    ↓
_handle_autodev_delegation()
    ↓
HermesAutoDevBridge.execute()
    ↓
HierarchicalExecutor.execute()
    ↓
┌─────────────────────────────────┐
│  1. Manager decomposes task     │
│  2. Coders implement subtasks   │
│  3. Reviewer validates code     │
│  4. Iterate if needed           │
└─────────────────────────────────┘
    ↓
Result aggregation
    ↓
Return to parent agent
```

### Role Mapping Implementation

| Phase | AutoDev Agent | Hermes Role | Wrapper Class |
|-------|---------------|-------------|---------------|
| Decomposition | ManagerAgent | plan | _HermesManagerWrapper |
| Implementation | CoderAgent | implement | _HermesCoderWrapper |
| Validation | ReviewerAgent | review | _HermesReviewerWrapper |

Each wrapper:
- Delegates to Hermes parent agent
- Maps role appropriately
- Provides async interface
- Handles errors gracefully

## Result Aggregation

Converts AutoDev's `HierarchicalResult` into Hermes-compatible format:

```python
{
    "results": [{
        "status": "completed",  # or "failed"
        "summary": "✓ Hierarchical execution completed...",
        "duration_seconds": 45.2,
        "autodev_mode": true,
        "files_modified": ["file1.py", "file2.py"],
        "iterations": 2,
        "review_iterations": 1,
        "agent_usage": {"manager-0": 1, "coder-0": 1, ...}
    }],
    "total_duration_seconds": 45.3,
    "autodev_mode": true
}
```
